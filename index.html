Control de versiones y Git

¿Qué es un control de versiones?

Es un sistema que registra cada cambio que se realiza en el código fuente de un proyecto. Esto te permite tener un histórico de todos los cambios producidos en sus ficheros, saber quién lo hizo y cuándo.

¿Qué es GIT?

Git es un software de control de versiones diseñado por Linus Torvalds, pensando en la eficiencia, la confiabilidad y compatibilidad del mantenimiento de versiones de aplicaciones cuando estas tienen un gran número de archivos de código fuente.

¿Qué es un repositorio?, repositorio locales y remotos.

Un repositorio remoto se aloja en un servidor remoto (puede estar en Internet o en un servidor externo; incluso puede ser el mismo equipo en una ruta diferente) y se comparte entre varios miembros del equipo. Un repositorio local se aloja en un equipo local para un usuario individual.

¿Qué es GITHUB?.

Un repositorio remoto se aloja en un servidor remoto (puede estar en Internet o en un servidor externo; incluso puede ser el mismo equipo en una ruta diferente) y se comparte entre varios miembros del equipo. Un repositorio local se aloja en un equipo local para un usuario individual.

¿Qué es una rama en GIT y para qué sirve?.

Usa una rama para identificar tareas de desarrollo sin afectar otras ramas en el repositorio. Cada repositorio tiene una rama por defecto y puede tener muchas otras ramas.

¿Cuáles son los estados en GIT?

Git tiene tres estados principales en los que se pueden encontrar tus archivos: confirmado (committed), modificado (modified), y preparado (staged).

¿Qué es la terminal, consola o línea de comandos y para qué sirve?

La ventana, que generalmente es llamada línea de comandos ó interfaz de línea de comandos, es una aplicación basada en texto para ver, manejar y manipular archivos en tu ordenador.

Funcionamiento de los comandos

ls:

Descripción: Lista el contenido del directorio actual.
Uso básico: ls
Ejemplo: Ejecutar ls en un directorio mostrará
todos los archivos y subdirectorios en ese directorio.

cd-directorio:

Descripción: Cambia el directorio de trabajo actual al especificado.
Uso básico: cd nombre_directorio
Ejemplo: cd documentos cambiará al directorio "documentos" dentro del directorio actual.

cd .. :

Descripción: Cambia el directorio de trabajo actual al directorio padre (el directorio que contiene al directorio actual).
Uso básico: cd ..
Ejemplo: Si estás en /home/usuario/documentos, ejecutar cd .. te llevará a /home/usuario.

pwd:

Descripción: Muestra la ruta completa del directorio de trabajo actual.
Uso básico: pwd
Ejemplo: Si estás en /home/usuario/documentos, ejecutar pwd mostrará /home/usuario/documentos.

mkdir-nombre:

Descripción: Crea un nuevo directorio con el nombre especificado.
Uso básico: mkdir nombre_directorio
Ejemplo: mkdir proyecto creará un nuevo directorio llamado "proyecto" en el directorio actual.

touch-nombre:

Descripción: Crea un nuevo archivo vacío o actualiza la fecha y hora de modificación de un archivo existente.
Uso básico: touch nombre_archivo
Ejemplo: touch archivo.txt creará un archivo vacío llamado "archivo.txt" si no existe, o actualizará su fecha y hora si ya existe.

rm-nombre:

Descripción: Elimina el archivo especificado.
Uso básico: rm nombre_archivo
Ejemplo: rm archivo.txt eliminará el archivo "archivo.txt".

cp-nombre-directorio:

Descripción: Copia el archivo especificado al directorio especificado.
Uso básico: cp archivo.txt /ruta/al/directorio/
Ejemplo: cp archivo.txt documentos/ copiará "archivo.txt" al directorio "documentos".

mv-nombre-directorio:

Descripción: Mueve el archivo especificado al directorio especificado. También se usa para renombrar archivos.
Uso básico: mv archivo.txt /ruta/al/directorio/
Ejemplo: mv archivo.txt documentos/ moverá "archivo.txt" al directorio "documentos". Si se proporciona un nuevo nombre en lugar de un directorio, también se puede usar para renombrar el archivo.
